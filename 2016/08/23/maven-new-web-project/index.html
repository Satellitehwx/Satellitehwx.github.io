在Eclipse中使用maven插件创建一个web project

发表于 2016-08-23 | 分类于 Maven |

  Maven项目对象模型(POM)，可以通过一小段描述信息来管理项目的构建，报告和文档的软件项目管理工具。
  Maven 除了以程序构建能力为特色之外，还提供高级项目管理工具。由于 Maven 的缺省构建规则有较高的可重用性，所以常常用两三行 Maven 构建脚本就可以构建简单的项目。由于 Maven 的面向项目的方法，许多 Apache Jakarta 项目发文时使用 Maven，而且公司项目采用 Maven 的比例在持续增长。——引自百度百科

Maven New Web-Project

首先创建一个Maven的Project如下图

首先创建工程

在New对话框Wizards中输入maven


点击Next 在 New Maven Project对话框中我们勾选上 Create a simple project (不使用骨架)

配置工程参数

  点击Next 进入下一对话框 在Group Id中填写好包路径 Artifact Id中写好相应的项目名 在Packaging中我们选择war


  点击Finish 创建项目 这时候 Package Explorer 中就会出现以Artifact Id 命名的项目，这个时候你会发现pom.xml报错了点开后有一个 “web.xml is missing and is set to true” 的错误 这是因为你创建的是web项目
而项目中没有web.xml文件这个时候我们 右击项目名在快捷菜单中选中Java EE Tools => Generate Deployment Descriptor Stub。
    这个时候就会在src-main-webapp-WEB-INF这个时候就会在src-main-webapp-WEB-INF 下面创建一个web.xml ，若此时创建了web.xml就不用执行下面创建WebContent目录的步骤(有的版本的eclipse是不会报错的)
由于packing是war包，那么下面也就多出了webapp的目录

配置工程目录

由于我们的项目要使用eclipse发布到tomcat下面，这里我们需要先把项目转成dynamic web project

在我们的项目上点击右键，选择properties 并找到 Project Facets

然后勾选Dynamic Web Module 并点击ok 如下图：(3.0只有tomcat7才支持)

接下来观察我们的项目结构，多了一个web content目录，如下图

  此时没有生成WebContent目录时： 我是把Dynamic web module 去掉勾选，然后ok，再点开项目的properties，再选中Dynamic web module就出来了。



虽然此时我们可以发布到tomcat中，但这不符合maven的结构，我们还要做如下修改
把上图WebContent下面两个目录 META-INF ，WEB-INF 直接剪切到src/main/webapp目录下，并删掉WebContent目录，那么现在的项目结构如下图：

    此时，pom.xml会报一个 “web.xml is missing and is set to true” 的错误因为我们移动了它自己生成的web.xml所以他会报错，我们再次右击项目名在快捷菜单中选中Java EE Tools => Generate Deployment Descriptor Stub 解决错误
然后我们要修改发布规则，右键点击项目， 选择 Deployment Assembly
选择WebContent，把它remove掉


接着重新指定一个web的路径，点击Add，选择Folder => Next


在src下找到webapp目录，然后Finish

修改Maven Build依赖

如果没有指定build path，我们需要把当前的build path 指向 Maven Dependency， 直接点击add，选择Java Build Path Entries 然后Next (如果已经存在Maven Dependency 则不需要此步骤)


然后再点击finish完成


完成后如下图：

至此一个基于maven的webapp就建立好了，并可以直接从eclipse中发布到tomcat中
补充：我们需要在src/main/webapp/WEB-INF下面创建一个web.xml (如果上面已经添加了web.xml就不用执行)

最后在pom中配置我们需要的依赖jar包
